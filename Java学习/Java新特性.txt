Java 7、8新特性
Java7语法新特性
	OpenJDK中的Coin项目的目的是维护对Java语言所做的语法增强。在Coin项目开始之初，曾向社区征求意见，一个月中收到70条意见，有9条列入考虑中，6条成为Java7的一部分，剩下两条在Java8中重新考虑。
	1、在switch中使用字符串，在Java之前，switch只能使用与整数兼容的类型
		实现原理：在编译器的层次实现，在编译过程中，利用优化策略将字符串转换成与整数兼容的格式。
					如果是一个case语句转换成if语句，一个case defaule语句转换成if-else语句，多个case转换成Java7之前的switch,用字符串的哈希值作为表达试的值
	2、数字字面量的改进
		字面量是指在源代码中直接表示一个固定的值。
		①二进制整数字面量，如0x表示16进制，Java7中可以用0b代表二进制
		②在数字字面量下使用下划线，比如不管整数还是浮点数，都允许在数字之间插入任意多个下划线。5_6.3_4。
	3、优化异常处理
		支持在catch语句中同时捕获多个异常，另一个是在捕获并重新抛出异常时的异常类型更加精确。
	4、try-with-resources语句，保证资源总被释放。
	5、优化变长参数的方法调用
		@SafeVarargs注解，在Java7之前只能使用@Suppress Warnings("unchecked")抑制编译器警告信息。
